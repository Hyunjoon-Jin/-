## 0. 한 줄 소개

플레이어가 축구 구단의 구단주가 되어 **선수 영입·육성**, **구단 경영**, **전술·경기 운영**을
모두 직접 다루며, 여러 시즌에 걸쳐 구단을 키워 나가는 **하드코어 시뮬레이션 게임**.

지향점은 Football Manager 계열의 깊이 있는 수치 기반 시뮬레이션이며,
PC 데스크톱에서 단일 플레이어로 즐기는 것을 1차 목표로 한다.

---

## 1. 확정된 핵심 의사결정

| 항목 | 결정 | 비고 |
|---|---|---|
| **핵심 재미** | 영입·육성 + 경영 + 전술 + 장기 육성 (4기둥 모두) | 어느 하나도 들러리가 아님 |
| **플랫폼** | PC 데스크톱 | Windows / macOS / Linux |
| **세션 형태** | 싱글플레이, 턴제(시즌·날짜 단위 진행) | 멀티는 추후 검토 |
| **시뮬 깊이** | 하드코어 (FM급 세부 스탯 + 분 단위 이벤트 시뮬) | |
| **데이터** | 가상 데이터 (절차적 생성) | 라이선스 리스크 회피 |
| **기술 스택** | TypeScript 모노레포 (엔진 분리 + Electron/React + SQLite) | 아래 6장 참고 |
| **개발 전략** | 수직 슬라이스 → MVP부터 점진 확장 | 얕지만 끝까지 도는 한 줄기 먼저 |

## 2. 디자인 기둥 (Design Pillars)

게임의 모든 기능은 아래 네 기둥 중 하나 이상을 강화해야 한다.
어느 기둥도 약화시키는 기능은 보류한다.

1. **영입·육성** — 스카우팅, 이적 시장, 유스 발굴, 선수 성장
2. **구단 경영** — 재정, 예산, 시설, 스폰서, 수입/지출 관리
3. **전술·경기 운영** — 포메이션, 전술 지시, 경기 중 개입
4. **장기 성장** — 다(多)시즌에 걸친 구단 가치 상승과 우승 도전

## 3. 도메인 모델 (개념 설계)

> 세부 능력치 목록과 공식은 별도 문서(`docs/engine.md`, 추후 작성)에서 확정한다.
> 여기서는 엔티티 구조와 관계만 정의한다.

### 3.1 핵심 엔티티

- **Player (선수)**
  - 식별: id, 이름, 국적, 생년(나이)
  - 능력치: 세부 스탯(공격/수비/피지컬/정신/골키핑 카테고리, 총 30~40개)
  - 잠재력(Potential): 범위형 — 환경/훈련에 따라 실현 정도가 달라짐
  - 포지션 적합도: 포지션별 숙련도
  - 상태: 컨디션, 사기(morale), 부상, 피로도
  - 계약: 소속 구단, 연봉, 계약 만료 시즌

- **Club (구단)**
  - 식별: id, 이름, 연고지, 소속 리그
  - 재정: 자금(balance), 주급 예산, 이적 예산
  - 평판/구단 가치
  - 시설: 훈련 시설, 유스 시설 등급
  - 소속 선수단(Squad), 전술(Tactic)

- **League (리그)**
  - 소속 구단 목록, 시즌 일정(Fixtures), 순위표(Table)
  - 승강제 규칙(추후), 컵대회(추후)

- **Match (경기)**
  - 홈/원정 구단, 날짜, 전술 스냅샷
  - 결과: 스코어, 이벤트 로그(분 단위), 선수별 평점

- **TransferOffer (이적 제안)**
  - 대상 선수, 제안 구단, 이적료, 상태

- **Tactic (전술)**
  - 포메이션, 라인별 지시(압박/템포/라인 높이 등), 선발/벤치

- **Staff (스태프)** — *추후 확장*
  - 코치, 스카우트, 의료팀 (선수 성장·부상 회복·스카우팅 정확도에 영향)

### 3.2 관계 요약

```
League 1 ─── * Club 1 ─── * Player
Club 1 ─── 1 Tactic
League 1 ─── * Match (Fixtures)
Match * ─── 2 Club (home/away)
Club * ─── * TransferOffer ─── 1 Player
```

### 3.3 시간 모델

- 게임은 **날짜 단위**로 진행한다(Continue 버튼).
- 시즌 = 리그 일정 + 이적 시장 창(window) + 시즌 종료 정산.
- 선수 나이/성장/계약은 시즌 경계에서 갱신된다.

## 4. 경기 엔진 (개념)

> 알고리즘 세부는 `docs/engine.md`에서 확정. 여기서는 방향성만 기록.

- **분 단위 이벤트 시뮬레이션**: 90분을 일정 틱으로 나눠 각 틱마다
  점유/전진/슈팅/세트피스 등 이벤트가 확률적으로 발생.
- 확률은 양 팀의 **전술 + 선수 능력치 + 컨디션/사기**의 함수.
- 출력: 스코어, 분 단위 이벤트 로그(텍스트 중계), 선수별 평점/스탯.
- **검증 가능성**: 엔진은 UI 없이 헤드리스로 실행 가능해야 하며,
  "N시즌 자동 시뮬 후 통계 분포 확인"으로 밸런스를 검증한다.
- 표현은 1차로 **텍스트 중계**, 추후 캔버스 기반 2D 경기뷰를 얹는다.

## 5. 진행 루프 & 메타게임

- **단기 루프**: 일정 진행 → 경기 → 결과/재정 반영 → 훈련/이적 결정.
- **시즌 루프**: 리그/컵 성적 → 상금·승강 → 이적 시장 → 다음 시즌 목표.
- **장기 목표**: 구단 가치 상승, 리그/컵 우승, (추후) 명예의 전당.
- **실패 조건**: 재정 파탄/성적 부진에 따른 압박 — 강도는 난이도로 조절.

## 6. 기술 스택 & 프로젝트 구조

### 6.1 스택

- **언어**: TypeScript (엔진·UI 공통)
- **시뮬 엔진**: 순수 TS, UI 의존성 0, 헤드리스 실행 + 단위 테스트
- **UI**: Electron + React (표·리스트·상세 패널 중심의 데이터 UI)
- **저장**: SQLite (better-sqlite3) — 세이브 파일 단위
- **테스트**: Vitest (엔진 로직 + 밸런스 시뮬)

### 6.2 모노레포 구조 (목표)

```
soccer-tycoon/
├─ packages/
│  ├─ engine/   # 순수 TS 시뮬레이션 코어 (선수/경기/이적/재정 + 테스트)
│  ├─ data/     # 가상 데이터 절차적 생성기 (선수·구단 수천 명)
│  └─ app/      # Electron + React 데스크톱 UI
├─ docs/        # 기획·설계 문서
└─ saves/       # SQLite 세이브 (런타임 생성)
```

### 6.3 선택 근거 요약

- UI 대부분이 표/리스트 → React가 가장 빠름.
- 엔진·UI 언어 통일 → 복잡한 도메인 타입을 양쪽이 공유, 타입 안전성 확보.
- 엔진 분리 → 헤드리스 밸런싱 검증 가능 (하드코어 시뮬의 생명).
- SQLite → 다시즌 대량 데이터를 파일 하나로 안정 저장·쿼리.

## 7. MVP 범위 (수직 슬라이스)

> 목표: "재미있는가?"를 가장 빨리 검증할 수 있는, 끝까지 도는 최소 게임.

### 7.1 포함 (MVP에 넣는 것)

- 단일 리그(예: 16~20개 구단), 1부 리그만.
- 가상 선수·구단 절차적 생성.
- 세부 능력치 스탯(축소판이라도 카테고리 구조 유지).
- 확률+이벤트 기반 경기 시뮬 + 텍스트 결과.
- 기본 전술(포메이션 + 선발 라인업).
- 기본 이적(이적 시장 창에서 영입/방출) + 단순 재정.
- 한 시즌 완주 → 순위표/우승 → 시즌 정산 → 다음 시즌 진입.
- SQLite 세이브/로드.

### 7.2 제외 (MVP 이후로 미루는 것)

- 다중 리그·국가, 승강제, 컵대회.
- 유스 시스템, 스태프(코치/스카우트), 시설 업그레이드.
- 2D 그래픽 경기뷰, 경기 중 실시간 개입.
- 스폰서/중계권 등 정교한 수입 모델.
- 멀티플레이.

### 7.3 MVP 화면 목록 (초안)

1. **대시보드** — 구단 요약, 다음 경기, 재정 한눈에.
2. **스쿼드** — 선수 목록/능력치/계약(정렬·필터).
3. **전술** — 포메이션 + 선발 지정.
4. **이적 시장** — 선수 검색/영입/방출.
5. **경기** — 진행/결과/이벤트 로그/평점.
6. **리그표** — 순위·일정·결과.

      양 팀 선발 11명을 포메이션 좌표에 점으로 배치(내 팀/상대 색 구분),
      공 위치에 따라 블록 이동·미세 스웨이. 하프타임 전술 변경도 반영.
      관전 중 실시간 통계 패널(점유율·슈팅·유효슈팅) — LiveMatch.stats().
- [x] 선수 사기·재계약: 출전 시간 기반 사기(핵심 선수 벤치 시 불만), 시즌 경계 갱신.
      선수 상세에서 사기·시즌 출전 표시 + 계약 만료 임박 선수 재계약(계약금·임금인상).
- [x] 경기 상세 통계: 점유율·슈팅·선수 평점(득점자 표시). 2D 관전 풀타임 +
      라운드 결과 클릭 → 상세 모달. (MatchResult의 기존 데이터 활용.)
- [x] 다중 리그·승강제: 1·2부(각 12팀, 총 24팀). 내 부는 경기 단위, 상대 부는
      자동 시뮬. 시즌 말 상하위 3팀 승강. 부별 목표(승격/잔류), 컵은 전 구단.
- [x] 선수 고유 특성(개성): 능력치·나이·포지션에 상관된 특성 8종(리더·유리몸·철강왕·
      특급유망주·골잡이·플레이메이커·다혈질·수비바위)을 생성 시 0~2개 부여. 파생 전력·
      부상·피로·성장·카드·오프시즌 사기에 반영. 선수 상세에 특성 칩 표시.
- [x] 경기 프리뷰(관전 전 스카우팅): 킥오프 대기 화면에 양 팀 리그 순위·최근 5경기 폼
      (승·무·패)·전력 7지표 비교 막대·핵심 선수를 표시. 엔진 form.ts로 폼 집계.
- [x] 국가대표 차출(international.ts): 시즌 사이 국적별 최상위 선수(최소 능력 이상) A매치
      차출. 캡(Player.caps) 누적·사기↑, 국제 경기 피로로 다음 시즌 컨디션 하락·부상 리스크.
      오프시즌 리셋 이후 적용. 선수 상세 캡 표시 + 대시보드 지난 시즌 차출/부상 요약.
- [x] 이적 협상: 즉시 영입 대신 제안-반응 협상. 호가=시장가×포지션 중요도(라인 서열)×
      잔여 계약. 제안≥호가=수락 / 하한~호가=역제안 / 하한 미만=거절. evaluateOffer(순수)
      + buyPlayerAt(타결 실행). UI는 협상 모달(제안 프리셋·역제안 수락).
- [x] 판매 입찰: 즉시 매각 대신 관심 구단들의 입찰(sellOffers) 중 선택. 입찰액=시장가×
      포지션 필요도(업그레이드·뎁스)×예산, 결정론적. acceptSellOffer로 실행. UI는 입찰 목록 모달.
- [x] 이사회 신뢰도·경질(board.ts): 시즌 종료 시 목표 대비 순위·승강·재정으로 신뢰도(0~100)
      변동. 하한(8) 미만이면 경질(게임 오버). 대시보드 신뢰도 미터·경고, 경질 화면. 난이도로
      목표가 달라져 실패 압박이 조절된다.
- [x] 이사회 특별 요구(demands.ts): 매 시즌 검증 가능한 요구(임금 감축·컵 우승·득점왕 배출)를
      상황 기반으로 생성. 시즌 종료 시 달성/실패로 신뢰도 가감(reward/penalty). 대시보드에 현재
      요구·지난 시즌 달성 여부 표시.
- [x] 컵 경기 관전: nextCupPairings(순수 대진)로 내 컵 경기를 리그처럼 2D 관전.
      playCupRound에 watched 결과 주입 — 내 경기는 관전 결과, 나머지는 시뮬. 하프타임 개입 가능.
      프리뷰(cupPreview)까지 리그와 동일하게 제공(타 부 상대는 순위·폼 미표시).
- [x] 선수 통산 기록: 경기 득점을 matchEffects에서 seasonGoals로 누적(리그+컵), 오프시즌에
      careerApps/careerGoals로 이월·시즌 카운터 리셋. 선수 상세에 시즌/통산 득점 표시.
      히스토리에 현역 통산 득점 순위(careerScorers, 전 구단) 명예의 전당 추가.
- [x] 선수 성장 곡선: 오프시즌마다 CA 스냅샷(Player.caHistory, 최근 20)을 쌓아 선수 상세에
      성장/노화 추이 SVG 스파크라인 표시(상승=녹색, 하락=주황, 시작·최고·현재 라벨).
- [x] 부상 등급 세분화(injury.ts): 부상 시 경미/중등도/중상 등급 + 부위명(Player.injuryName)
      부여. 의료 스태프가 경미 비중·회복 기간을 완화. 선수 상세/스쿼드에 부상명 표시.
- [x] 선수 최근 폼(recentPlayerForm): 진행 중 시즌 결과에서 특정 선수의 최근 n경기 평점을
      추출. 선수 상세에 평점 칩(색상 등급) + 상대·홈/원정 툴팁으로 표시.
- [x] 역대 우승 선수단 스냅샷(seasonSquadSnapshot): 리그·컵 우승 시즌의 선발 라인업을
      (포지션·이름·나이·평균 평점·득점)으로 캡처 — 오프시즌(나이 증가) 전에 스냅샷해
      "그 시즌 당시" 기록 보존. 히스토리 역대 시즌 표에서 우승 시즌에 "🏆 스쿼드" 버튼으로 열람.
- [x] 경기 중 부상 이벤트 + 긴급 교체: 부상 판정을 matchEffects(경기 후)에서
      simulateMatch.generateInjuries(킥오프 시점, 카드와 동일 패턴의 독립 시드)로 이전 —
      LiveMatch.injuries()로 언제든 조회 가능해 관전 중 분 단위로 실시간 노출할 수 있다.
      내 선수가 부상당하면 진행을 멈추고 벤치 중 교체할지 묻는 모달을 띄운다(즉시
      live.setTactic 반영, 계속 뛰게 두는 선택도 가능). matchEffects는 결과에 실린
      판정을 그대로 적용만 하도록 리팩터(단일 소스 유지 — 관전 중 본 스케줄과 최종
      반영 결과가 항상 일치).
- [x] 스카우팅 리포트(scouting.ts): CA→전체 등급(월드클래스~주전 경쟁 필요), PA-CA 갭+나이→
      잠재력 등급(스카우팅 레벨 낮으면 미상, 30세 이상은 갭 무관 제한적 취급), 나이 프로필
      (유망주/전성기/베테랑/노장), 포지션 관련 능력 중 강점·약점 3개씩. 엔진은 분류만
      산출하고 한국어 문구 조립은 UI(PlayerDetail)에서 담당 — 선수 상세에 리포트 패널로 표시.
- [x] 은퇴 선수 레전드 아카이브: runOffseason이 은퇴(37세 이상) 선수를 스쿼드에서 제거하기
      직전에 통산 기록(RetiredLegend: 이름·포지션·은퇴 나이·통산 출전/득점·A매치 캡)을
      캡처해 반환. 앱은 내 구단 소속 은퇴자만 GameState.legends에 영구 누적(은퇴 시즌 포함).
      히스토리에 "🕯️ 레전드 명예의 전당" 표로 표시 — 우승 스쿼드 스냅샷·통산 득점 순위와
      함께 명예의 전당을 완성.
- [x] 레전드 은퇴식 배너: 대시보드 "지난 시즌" 패널에서 game.legends를 그 시즌 번호로
      필터링해, 내 구단 선수가 은퇴한 시즌이면 통산 기록(출전·득점·A매치)과 함께
      헌정 배너를 강조 표시. 엔진 변경 없이 앱 레이어에서 기존 legends 데이터를 재사용.
- [x] 통산 마일스톤 알림: runOffseason이 통산 기록 누적 직전/직후 값을 비교해 이번
      시즌에 처음 넘은 임계값(출전 50/100/150/200/250/300, 득점 10/25/50/100/150/200)을
      CareerMilestone으로 기록(SeasonSummary.milestones, 앱 전용). 대시보드에 초록색
      축하 배너로 표시. 한 시즌에 여러 임계값을 동시에 넘으면 모두 기록.
- [x] 라이벌 구단(더비 매치): 엔진 변경 없이 앱 레이어(game.ts)에서 처리. 게임 시작
      시 같은 부에서 평판(finance.reputation)이 가장 가까운 구단을 라이벌로 1회 고정
      (승강으로 부가 바뀌어도 그대로 유지 — "창단 라이벌" 내러티브). 시즌 종료
      시점에 이번 시즌 라운드로빈 결과(ss.results)에서 나-라이벌 맞대결만 걸러
      승무패를 GameState.rivalRecord에 누적(다른 부일 땐 대결이 없어 전적 변화 없음).
      대시보드에 라이벌 카드(구단명·부·통산 전적), 경기 탭 다음 경기·관전 프리뷰에
      "🔥 라이벌전" 배지/배너로 표시. 특별한 경기력 보정은 없음(추적·연출 전용).

기획했던 핵심·확장 시스템이 모두 구현됨. 이후는 콘텐츠 확충·디테일 다듬기 영역.

## 9. 미해결 결정 사항 (열린 질문)

기획을 더 진행하기 전 또는 진행하며 확정해야 할 항목:

1. 세부 능력치를 **정확히 몇 개, 어떤 항목**으로 둘 것인가?
2. 경기 시뮬의 **틱 단위**(분/이벤트)와 핵심 변환식.
3. 이적료·연봉 **산정 공식**의 기준값.
4. MVP 리그 **규모**(구단 수, 시즌 경기 수).
5. 난이도/실패 조건의 **강도**.
