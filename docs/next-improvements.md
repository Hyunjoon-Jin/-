## 0. 현재 구현 진단 — 무엇이 비어 있나

`packages/engine/src`를 훑어 확인한 **미구현/골격뿐인 영역**:

| 영역 | 현황 | 공백 |
|---|---|---|
| 선수 심리 | `morale`(0~1 스칼라)만 존재 | 출전시간 불만·이적요청·약속·드레싱룸 파벌이 전무 |
| 감독 커리어 | 이사회 신뢰도만 있음 | 감독 평판·타 구단 잡오퍼·경질 후 재취업·커리어 이력 없음 |
| 팬·티켓 | `finance`에 관중수·매치데이 수익만 | 티켓 가격 정책·시즌권·울트라스·굿즈가 없음 |
| 스쿼드 규정 | 인원 상한만 | 홈그로운 쿼터·외국인 제한·컵 등록 명단 규정 없음 |
| 유스 | 시즌말 기대주 소개 | 인테이크 데이 이벤트·리제너레이션 등급·성격 생성 없음 |
| 경기 심화 | 관전·이벤트 풍부 | PK 슛아웃 상호작용·세트피스 루틴 지정·VAR 판정 없음 |
| 데이터 | 표 위주 통계 | 히트맵·xG 타임라인·패스맵·레이더 차트 없음 |
| 세이브/QoL | 단일 세이브 | 다중 슬롯·명시적 오토세이브 UI·설정 화면·되돌리기 없음 |

이 8개 축을 중심으로 55건을 도출한다.

## 1. 개선 항목 (총 55건)

우선순위: ★★★ = 게임에 새로운 재미 축을 여는 핵심 · ★★ = 체감 큰 확장 · ★ = 완성도

## 2. 구현 로드맵 제안

각 페이즈는 독립 배포 가능하며, 앞선 세션들의 리거 사이클(엔진 로직+테스트 → 앱 UI → 스크린샷 검증 → PR·머지)을 그대로 따른다.

| 페이즈 | 범위 | 여는 재미 |
|---|---|---|
| **P1 — 드레싱룸** | A1·A2·A3·A10 (+ A4·A8) | "선수를 관리한다"는 새 축의 핵심 루프 |
| **P2 — 감독 커리어** | B1·B2·B3·B4 | 경질=게임오버를 넘어선 장기 커리어 동기 |
| **P3 — 유스 인테이크** | E1·E2·E3·E4 | 매 시즌 기대되는 이벤트 스파이크 |
| **P4 — 팬·티켓 경제** | C1·C2·C3 | 경영 결정의 새 레버 |
| **P5 — 스쿼드 규정** | D1·D2·D5 | 스쿼드 구성의 전략적 제약 |
| **P6 — 데이터 시각화** | G1·G2·G3·G4 | 기존 데이터의 분석 재미 |
| **P7 — 경기 심화** | F1·F2·F4 | 시뮬의 마지막 상호작용 조각 |

P1(드레싱룸)과 P2(감독 커리어)가 게임에 가장 큰 **새로운 축**을 열므로 최우선.

---

## 3. 설계 원칙 (구현 시 준수)

1. **엔진 결정성·재현성 유지** — 새 시스템도 시드 기반 결정성을 깨지 않는다.
   심리·이벤트 판정은 기존 RNG 스트림을 재사용하고, 세이브 하위호환(누락 필드
   기본값)을 지킨다.
2. **스칼라 남발 금지** — 행복도·평판 등 새 스칼라는 "왜 오르내리는지"가 UI에서
   설명돼야 한다(입력 요인 노출). 숨은 숫자는 불신을 낳는다.
3. **기존 시스템과 접합** — 새 축은 고립되면 안 된다. 행복도는 이적·재계약·출전에,
   감독 평판은 협상·잡오퍼에, 유스는 임대·스카우트에 반드시 연결한다.
4. **되돌릴 수 없는 결정엔 무게** — 이적 요청 수락, 잡오퍼 수락, 유스 방출 등은
   확인 절차와 결과 예고를 둔다(기존 확인 모달 패턴 재사용).
5. **정보 → 판단 → 행동 → 피드백** — 대시보드 경고(정보) 없이 심리 시스템만 넣으면
   "안 보이는 불만"이 된다. 각 축은 대시보드 노출부터 함께 만든다.
