# 경기 관전 — 페이싱 & 실시간 전술 개입 개선안

> 작성일: 2026-07-10
> 상태: 아이디어 도출 완료 (구현 전)
> 배경: 두 가지 핵심 불만 — **(1) 경기 시뮬레이션이 너무 빠르게 진행된다**,
> **(2) 경기 중 감독이 전술을 수정하거나 개입할 수 있는 여지가 사실상 없다.**

---

## 0. 현재 구현 진단

### 페이싱(진행 속도) 현황

| 항목 | 현재 값 | 문제 |
|---|---|---|
| 틱 주기 | `TICK_MS = 130ms`에 경기 1분 진행 | **1x 배속에서 90분 경기가 약 12초 만에 끝난다** |
| 배속 | 1x / 2x / 4x (더 빠르게만 가능) | 천천히 볼 방법이 아예 없음 |
| 진행 단위 | `LiveMatch.runUntil(minute)` — 분 단위 | 분 사이의 전개(빌드업·역습 과정)가 화면에 존재하지 않음 |
| 이벤트 연출 | 골 배너 1.5초, 카드/부상 아이콘 2초 | 배너가 떠 있는 동안에도 경기는 그대로 질주 |
| 스킵 | 하프타임까지 / 종료까지 2종 | 중간 단위(5분, 다음 이벤트까지)가 없음 |

결론: **"관전"이라기보다 결과 애니메이션에 가깝다.** 경기를 읽고 → 판단하고 →
개입하는 감독 루프가 성립하려면 1x가 최소 수 분짜리 경험이어야 한다.

### 전술 개입 현황

| 개입 수단 | 가능 시점 | 한계 |
|---|---|---|
| 전술 슬라이더·포메이션·개인지시 편집 | **하프타임 한정** | 인게임에는 편집 UI 자체가 안 뜸 |
| 퀵 지시 (추격 모드 / 리드 지키기) | 지고/이기고 있을 때만 | 동점이면 아무 프리셋도 없음, 2종뿐 |
| 교체 (3회) | 인게임 상시 (모달·벤치 드래그) | 유일하게 온전한 인게임 개입 수단 |
| 일시정지 / 배속 / 스킵 | 인게임 상시 | 시간 제어일 뿐 전술 개입은 아님 |

엔진은 `LiveMatch.setTactic(side, tactic)`을 **경기 중 언제든** 받을 수 있는데,
UI가 하프타임에만 열어주고 있다. 즉 병목은 엔진이 아니라 **관전 UI 설계**다.

## 3. 설계 원칙 (구현 시 준수)

1. **일시정지는 무료, 시간은 유한** — 생각은 얼마든지(pause), 경기 내 개입은
   쿨다운·횟수로 무게를 준다(B11). "슬라이더 스팸"이 최적해가 되면 안 된다.
2. **엔진은 이미 준비돼 있다** — `setTactic`은 분 단위 어느 시점이든 받는다.
   UI 개입 지점을 늘리는 것이 본체이고, 엔진 확장(F군)은 표현력 보강이다.
3. **재현성 유지** — 부상·카드 스케줄이 킥오프 시점 확정인 기존 설계를 깨지 않는다.
   개입은 이후 전개 확률에만 영향을 주고, 같은 개입 = 같은 결과여야 한다(F5).
4. **정보 → 판단 → 개입 → 피드백 루프** — C군(정보) 없이 B군(개입)만 늘리면
   "뭘 눌러야 할지 모르는 버튼들"이 된다. 페이즈 순서가 이를 반영한다.
